# AT82.08 Computer Vision
## AI-Generated Image Classification

### Setting Up the Environment
To set up the environment for this project, run these commands:
```bash
git clone <repository_url>
cd <repository_name>
uv sync
```

### File Structure
- `data/`: Directory containing the dataset of images from original source (Kaggle).
  - `train/`: Training images. (Git Ignore this folder)
    - `FAKE/`: Images generated by AI.
    - `REAL/`: Authentic images.
  - `test/`: Testing images. (Git Ignore this folder)
    - `FAKE/`: Images generated by AI.
    - `REAL/`: Authentic images.
  - `<*_features_train>.csv`: CSV files containing extracted features for training images.
  - `<*_features_test>.csv`: CSV files containing extracted features for testing images.
- `outputs/`: Directory for saving model outputs and results.
  - `models/`: Saved trained models from ML experiments.
  - `models_deep/`: Saved metrics and trained models from Deep Learning experiments.
- `utils/`: Utility scripts for data processing and feature extraction.

### Notebooks for Feature Extraction & Analysis
- `color_analysis.ipynb`: Notebook for analyzing color distributions in images.
- `lbp_analysis.ipynb`: Notebook for analyzing Local Binary Patterns (LBP) in images.
- `gabor_analysis.ipynb`: Notebook for analyzing Gabor filter responses in images.
- `hog_analysis.ipynb`: Notebook for analyzing Histogram of Oriented Gradients (HOG) in images.

### Notebooks for Experiments
- `ml_training.ipynb`: Notebook for training and evaluating traditional machine learning models.
- `cnn_training.ipynb`: Notebook for training and evaluating Convolutional Neural Networks (CNNs).

### Notebooks for Visualizations and Evaluation
- `evaluation.ipynb`: Notebook for evaluating model performance and visualizing results.
- `plot_sample_algo.ipynb`: Notebook for plotting sample outputs from different algorithms to be used in the report and presentation.
